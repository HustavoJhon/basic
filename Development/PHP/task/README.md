# RÚBRICA DE EVALUACIÓN DEL TRABAJO PRÁCTICO GRUPAL  
## INFORME DE LA FUNCIONALIDAD DE LA PÁGINA WEB Y CÓDIGO FUENTE  
**Unidad 2 - Semana 7**  

**Asignatura:** Programación Web  

---

## Descripción
Cada equipo deberá implementar un sistema de inicio de sesión en una aplicación web. El objetivo es que los estudiantes puedan definir claramente la estructura **Modelo, Vista, Controlador (MVC)** utilizando PHP puro y entregar un informe que documente el funcionamiento de la página web, el código fuente y la base de datos.

---

## Consigna

### 1. Desarrollo de un sistema de inicio de sesión
- Implementar un inicio de sesión donde se verifique si el usuario está activo.
- Debe existir la opción de cerrar sesión.
- El usuario debe estar registrado en una base de datos.
- Crear un CRUD de usuarios.

---

### 2. Implementación de la funcionalidad de la página web
- Crear un carrito de compras funcional que permita:
  - Agregar productos
  - Quitar productos
  - Vaciar carrito
  - Realizar compra (guardada en base de datos)
- Implementar un CRUD de productos accesible solo para administrador.

---

### 3. Documentación del código fuente
- Explicar la arquitectura MVC (Modelo, Vista, Controlador).
- Definir los componentes y sus responsabilidades.
- Código bien comentado y organizado siguiendo buenas prácticas.

---

### 4. Evaluación de la funcionalidad
- Incluir análisis de fortalezas y mejoras.
- Agregar capturas de pantalla del sistema funcionando.

---

## Rúbrica de Evaluación

| Criterio | Insuficiente (0) | Suficiente (1) | Notable (2) | Sobresaliente (3) |
|----------|------------------|----------------|-------------|-------------------|
| **Funcionalidad de la página web** | No funciona o tiene errores graves | Funciona con algunos errores | Funciona bien con pequeños detalles | Funciona perfectamente sin errores |
| **Calidad y organización del código** | Desorganizado, con errores graves | Funcional pero poco claro | Bien organizado y buenas prácticas | Código limpio, optimizado y estructurado |
| **Inicio de sesión, carrito y CRUD** | No implementado o incompleto | Implementado con errores | Bien implementado | Implementación completa, eficiente y escalable |
| **Documentación del informe** | Incompleto o mal redactado | Explica lo básico | Bien estructurado | Completo, detallado y claro |
| **Uso del MVC** | No definido | Definido de forma general | Bien definido | Perfectamente definido y justificado |
| **Interfaz de usuario (UI)** | Deficiente y desordenada | Funcional pero poco atractiva | Clara y usable | Intuitiva, atractiva y optimizada |
