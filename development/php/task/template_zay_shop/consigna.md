# RÚBRICA DE EVALUACIÓN DEL TRABAJO PRÁCTICO GRUPAL  
## INFORME DE LA FUNCIONALIDAD DE LA PÁGINA WEB Y CÓDIGO FUENTE  
**Unidad 2 - Semana 7**  

**Asignatura:** Programación Web  

---

## Descripción

Cada equipo deberá implementar un sistema de inicio de sesión en una aplicación web. El objetivo es que los estudiantes puedan definir claramente la estructura **Modelo, Vista, Controlador (MVC)** utilizando el lenguaje PHP puro y entregar un informe que documente el funcionamiento de la página web desarrollada, el código fuente asociado y la base de datos.

---

## Consigna

### 1. Desarrollo de un sistema de inicio de sesión

- Implementar un inicio de sesión donde se verifique que el usuario está activo.
- Debe incluir la opción de cerrar sesión.
- El usuario debe estar registrado en una base de datos.
- Crear un CRUD para los usuarios del sistema.

---

### 2. Implementación de la funcionalidad de la página web

- Crear una página web con un carrito de compras funcional que permita:
  - Agregar productos  
  - Quitar productos  
  - Vaciar carrito  
  - Realizar la compra (guardando los datos en la base de datos)

- Implementar un CRUD de productos accesible solo para el administrador.

---

### 3. Documentación del código fuente

- Explicar la arquitectura MVC (Modelo, Vista, Controlador).
- Definir los componentes y sus responsabilidades.
- Asegurar que el código esté bien documentado y siga buenas prácticas.

---

### 4. Evaluación de la funcionalidad

- Incluir un análisis de fortalezas y áreas de mejora.
- Presentar capturas de pantalla del sistema en funcionamiento.
- Incluir evidencia de la correcta implementación.

---

## Rúbrica de Evaluación

| Criterio | Insuficiente (0) | Suficiente (1) | Notable (2) | Sobresaliente (3) |
|----------|------------------|----------------|-------------|-------------------|
| **Funcionalidad de la página web** | No funciona o tiene errores graves | Funciona con algunos errores | Funciona correctamente con pequeños detalles | Funciona perfectamente y cumple todos los requisitos |
| **Calidad y organización del código fuente** | Código desordenado y con errores graves | Funcional pero poco claro | Bien organizado y con buenas prácticas | Código limpio, optimizado y bien estructurado |
| **Inicio de sesión, carrito y CRUD** | No implementado o incompleto | Implementado con errores | Bien implementado | Implementación completa, eficiente y escalable |
| **Documentación del informe** | Incompleto o mal redactado | Explica lo básico | Bien estructurado | Completo, detallado y claro |
| **Uso del MVC** | No definido | Definido de forma general | Bien definido | Perfectamente definido y justificado |
| **Interfaz de usuario (UI)** | Deficiente y desorganizada | Funcional pero poco atractiva | Clara y usable | Intuitiva, atractiva y optimizada |
